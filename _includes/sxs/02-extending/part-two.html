Part Two Customizing card using custom styles

Using custom styles, we can customize our components beyond theme settings.

Add a custom class for our new variant.

We'll use this class to identify our custom card component and assign custom styles.

When adding custom classes we recommend using a two- to three-letter prefix for clear identification. USWDS primarily uses usa-. Since this is your tutorial, lets use my-card as our custom class.

description _includes/card.html

<div class="usa-card">
<div class="usa-card my-card">
 <div class="usa-card__container">
  <div class="usa-card__header">

Open a new terminal window.

At this point you should already have two terminal windows open. One is serving our site and the other is compiling our stylesheets. We're going to open one more so we can add a couple new files.

Open a new terminal window and navigate to the tutorial's root directory, as before. In the following example, change path/to/uswds-tutorial to the actual path. 

cd path/to/uswds-tutorial

Create a custom stylesheet for our custom card component.

To help keep things organized, we'll create a custom stylesheet with styles specific to our customized card component. In later exercises, we'll use this same stylesheet to extend the card component into a new custom component. First, let's add a components directory to our css assets. 

mkdir _theme/components

Now let's add our new my-card stylesheet. It's standard naming convention to prefix SCSS partials with an underscore (as we've already seen in _uswds-theme and _uswds-theme-custom-styles). 

touch _theme/components/_my-card.scss

Import the new stylesheet into USWDS custom styles.

Navigate to _theme/_uswds-theme-custom-styles.scss and import our new stylesheet into the USWDS custom styles file.

description _theme/_uswds-theme-custom-styles.scss

@use "uswds-core" as *;
@forward "../assets/css/styles.scss";

@forward "components/_my-card.scss";

Once you've updated this file, you can save and close it. Since we've added our project Sass source directory to our gulpfile in the first part of the tutorial, it should already be compiling correctly!

Now that our stylesheet is being imported, lets write some customs styles to finish up our card.

Use USWDS core features in custom styles.

We'll want to use USWDS design tokens in our stylesheets using USWDS style features like functions and mixins. To do this, lets add uswds-core to the top of our new stylesheet with the use Sass directive.

description _theme/components/_my-card.scss

@use "uswds-core" as *;

Add custom class and styles.

Next lets create a custom class for our personalized component and use some USWDS utilities. We'll start by updating the background color of the card. We can use the color() function to use color tokens in our CSS rules.

description _theme/components/_my-card.scss

.my-card .usa-card__container {
  background-color: color("yellow-10");
}

After recompiling we can see our changes have taken effect. Let's refine this card a bit more by reducing the boldness of the header and adding bit more space before the call to action. We'll use units tokens and font weight tokens for this.

description _theme/components/_my-card.scss

.my-card .usa-card__container {
  background-color: color("indigo-70v");
}


.my-card .usa-card__footer {
    margin-top: units(1);
}

.my-card .usa-card__heading {
    font-weight: font-weight("bold");
}

Now we're making progress. This component is looking pretty good.
